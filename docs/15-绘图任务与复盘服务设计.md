# 15. 绘图任务与复盘服务设计

## 15.1 设计目标

复盘绘图服务负责把编辑结果、前后 24h 数据和 IFS 诊断场组织成可复现的复盘产品。绘图失败不得影响版本保存；缺失变量应允许 partial_success。

---

## 15.2 三层架构

```text
第一层：原始参考脚本
reference/要素场绘图/*.py
仅保留绘图逻辑、色标、版式参考，不由业务直接调用。

第二层：plotter 纯函数
plotters/*.py
只接收数组、坐标、配置、output_path，不读数据库、不判断业务状态。

第三层：review_plot_service
services/review_service.py / plot_task_service.py
读取数据库与 manifest，调度 plotter，记录状态和日志。
```

---

## 15.3 plotter 函数规范

所有 plotter 必须满足：

```text
1. 不读全局变量；
2. 不自行判断当前起报时间；
3. 不硬编码输入/输出路径；
4. 不写数据库；
5. 输入缺失时抛出明确异常；
6. 返回 output_path。
```

示例：

```python
def plot_precip_product(
    qpf: np.ndarray,
    ptype: np.ndarray,
    grid_def: dict,
    product_type: str,
    metadata: dict,
    boundary_config: dict,
    output_path: str,
) -> str:
    ...
```

```python
def plot_circulation_panel(
    hgt_data: np.ndarray,
    wind_u: np.ndarray,
    wind_v: np.ndarray,
    rh_data: np.ndarray,
    lon: np.ndarray,
    lat: np.ndarray,
    level: int,
    init_time: str,
    lead_hour: int,
    region: str,
    boundary_config: dict,
    output_path: str,
) -> str:
    ...
```

## 15.4 复盘模板

模板配置：

```json
{
  "template_id": "snow_phase_review_v1",
  "template_name": "雨雪相态订正复盘图",
  "required_fields": ["qpf_before", "ptype_before", "qpf_after", "ptype_after", "delta_qpf", "change_ptype"],
  "optional_fields": ["z500", "t850", "rh700", "u850", "v850"],
  "allow_partial_success": true,
  "panels": [
    {"id": "before", "type": "precip_phase", "fields": ["qpf_before", "ptype_before"]},
    {"id": "after", "type": "precip_phase", "fields": ["qpf_after", "ptype_after"]},
    {"id": "delta", "type": "delta_qpf", "fields": ["delta_qpf"]},
    {"id": "synoptic", "type": "circulation", "fields": ["z500", "u850", "v850", "rh700"]}
  ]
}
```

模板字段：

| 字段 | 说明 |
|---|---|
| required_fields | 缺失时任务 failed，除非模板允许降级 |
| optional_fields | 缺失时记录 missing_fields，面板可跳过 |
| allow_partial_success | 是否允许部分面板跳过 |
| panels | 复盘图面板定义 |

## 15.5 review_payload

服务层生成：

```json
{
  "metadata": {
    "case_id": "2026010108",
    "window_id": "2026010108_ACC24_024_048",
    "version_id": "v003",
    "init_time": "2026-01-01T08:00:00Z",
    "accum_hours": 24,
    "start_lead": 24,
    "end_lead": 48,
    "grid_id": "region_005deg_v1"
  },
  "edit_fields": {},
  "review_windows": {},
  "ifs_fields": [],
  "missing_fields": [],
  "template": {},
  "output": {},
  "plot_task": {}
}
```

规则：

```text
1. review_payload 是复盘图可复现的关键文件，必须归档。
2. payload 中路径使用相对路径，根目录为 review_root。
3. 所有缺失字段记录到 missing_fields。
```

## 15.6 当前窗口与前后 24h

```text
current_window = [start_lead, end_lead]
prev24_window = [start_lead - 24, start_lead]
next24_window = [end_lead, end_lead + 24]
```

边界：

| 场景 | 处理 |
|---|---|
| prev24 start < 0 | prev24 标记 missing，不报错 |
| next24 end > max_lead | next24 标记 missing，不报错 |
| 对应 qpf/ptype 缺失 | missing_fields |

## 15.7 IFS 变量取样

配置：

```json
{
  "review_time_policy": {
    "current_window": ["middle", "end"],
    "prev24_window": ["end"],
    "next24_window": ["start"]
  }
}
```

取样函数：

```python
def resolve_lead_hours(start_lead: int, end_lead: int, policy: list[str]) -> list[int]:
    ...
```

示例：

```text
start=24,end=48, policy=[middle,end] → [36,48]
```

如果 IFS 只有 6h 步长，需使用最近可用时效或在配置中声明：

```json
{"ifs_time_step_hours": 6, "lead_rounding": "nearest"}
```

## 15.8 绘图任务队列

状态机：

```text
pending → running → success
pending → running → partial_success
pending → running → failed → pending(retry) → permanently_failed
```

队列配置：

```json
{
  "plot_queue": {
    "max_concurrent": 2,
    "max_retries": 3,
    "retry_delay_seconds": 30,
    "task_timeout_seconds": 300
  }
}
```

任务执行流程：

```text
1. ReviewService 创建 review_product(status=pending, attempt=0)
2. PlotTaskService 入队
3. Worker 认领任务：UPDATE review_product SET locked_by=worker_id, locked_at=now(), status='running', attempt=attempt+1 WHERE status='pending' AND (next_retry_at IS NULL OR next_retry_at <= now()) AND locked_by IS NULL ORDER BY created_at LIMIT 1
4. 读取 review_payload
5. 遍历 panels
6. 对缺失 optional 字段的 panel 写缺失占位图
7. 生成组合图
8. 成功：status='success'/'partial_success', locked_by=NULL
   失败：若 attempt < max_retries → status='failed', locked_by=NULL, next_retry_at=now()+retry_delay
         若 attempt >= max_retries → status='permanently_failed', locked_by=NULL
```

Worker 恢复策略（服务启动时）：

```text
1. 扫描 status='running' 且 locked_at < now() - task_timeout_seconds 的记录，视为僵死任务
2. 僵死任务：若 attempt < max_retries → status='failed', locked_by=NULL, next_retry_at=now()
             若 attempt >= max_retries → status='permanently_failed', locked_by=NULL
3. 扫描 status='failed' 且 next_retry_at <= now() 的记录，重新入队
```

## 15.9 缺失变量处理

`missing_fields` 项：

```json
{
  "name": "rh",
  "level_type": "pressure",
  "level_value": 700,
  "lead_hour": 72,
  "reason": "file_not_found",
  "expected_path": "..."
}
```

处理：

| 缺失类型 | 处理 |
|---|---|
| optional field 缺失 | 面板跳过或显示占位，status=partial_success |
| required field 缺失 | 任务 failed，除非模板声明 fallback |
| dimension_mismatch | 任务 failed |
| read_error | 可重试，失败记录 traceback |

占位图：

```text
灰底 + “数据缺失：rh 700hPa +072h”
```


## 15.11 绘图日志

每个 review_id 必须有 `plot_log.txt`：

```text
[time] task start
[time] load payload
[time] panel before success
[time] panel synoptic skipped: missing rh700 +072h
[time] composite success
[time] task partial_success
```

异常需记录 traceback。

## 15.12 绘图测试

测试用例：

```text
1. 全字段齐全 → success
2. optional 字段缺失 → partial_success
3. required 字段缺失 → failed
4. 单 panel 绘图异常 → failed 或 partial_success，按模板配置
5. 超时 → failed 并可 retry
6. retry 成功 → success
7. 导出复盘包包含完整 manifest 和日志
```
