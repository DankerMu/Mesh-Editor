### 10.3.1 网格渲染图层设计（PrecipPhaseGridLayer）

参考项目 `yy-webgis-bj` 中 `GirdValueWebGLTileLayer.js` 已实现 WebGL 栅格渲染、DataTileSource、鼠标拾取等能力，可借鉴其 tile loader 和 debug overlay 机制，但**不能直接复用其颜色映射逻辑**。

#### 不能直接复用的原因

| 差异 | GirdValueWebGLTileLayer | 本系统需求 |
|---|---|---|
| 输入 | 单变量 data 数组 | qpf 数组 + ptype 数组（双变量） |
| 色标 | 单一 values/colors 线性分级 | ptype 决定色标函数，qpf 决定色级 |
| 经度处理 | `((lon % 360) + 360) % 360` 全球 wrapX | 固定区域 70-111°E，不需要 wrapX |
| 鼠标拾取 | 返回单个数值 | 需返回 qpf + ptype + 编辑状态 |

#### 新建 PrecipPhaseGridLayer

```text
输入：
  qpf_array    — Float32Array, 501×821, 订正后累计降水
  ptype_array  — Uint8Array,   501×821, 订正后相态编码
  color_config — 色标配置对象（来自 §9.2 色标定义）
  grid_def     — grid_definition.json 中的网格参数

输出：
  RGBA tile（通过 DataTileSource 提供给 WebGLTileLayer 渲染）
```

#### 像素着色逻辑

遵循 `GribToPng.java` 的 `pngCombinePtype()` 方法（参考代码 `reference/cdsz/.../GribToPng.java:197-288`）：

```text
对每个格点 (i, j)：
  qpf   = qpf_array[i][j]
  ptype = ptype_array[i][j]

  如果 qpf < 0.1 或 ptype == 0：
    RGBA = (0, 0, 0, 0)            // 透明

  如果 ptype == 2（雪）：
    RGBA = snow_color_lookup(qpf)   // 灰色分级

  如果 ptype == 3（雨夹雪）：
    RGBA = (251, 201, 252, 255)     // 固定粉紫

  否则（ptype == 1，雨）：
    RGBA = rain_color_lookup(qpf)   // 绿蓝紫分级
```

#### 坐标约束

- `getDataValue(lon, lat)` **删除** `((lon % 360) + 360) % 360` 归一化逻辑。
- 经纬度严格钳位到 `[lon_min, lon_max]` × `[lat_min, lat_max]`，超出范围返回 null。
- 选区 mask、鼠标拾取同样钳位，确保所有交互坐标不溢出固定区域。

#### 鼠标拾取返回值

```text
{
  lon, lat,
  grid_i, grid_j,
  qpf_before, qpf_after,
  ptype_before, ptype_after,
  is_edited
}
```

#### 可复用的参考代码部分

| 参考代码 | 可复用部分 |
|---|---|
| GirdValueWebGLTileLayer.js | DataTileSource 框架、tile 缓存机制、WebGL style 配置、updateData 刷新流程 |
| GirdValueWebGLTileLayer.js | debug overlay 鼠标跟随显示 |
| CustomTileLayer.js | Canvas fallback 方案（WebGL 不可用时降级） |

#### 降水调整

```text
定值：qpf = value
增加：qpf += delta
减少：qpf -= delta，最小为 0
乘系数：qpf *= factor
清零：qpf = 0，ptype = 0
弱降水筛查：qpf <= threshold 时清零
```

#### 相态调整

```text
设为无降水（ptype=0）
设为雨（ptype=1）
设为雪（ptype=2）
设为雨夹雪（ptype=3）
恢复原始相态
```
