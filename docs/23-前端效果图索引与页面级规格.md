## 23.3 页面级规格

### F01 网格编辑主界面（已生成）

- 路由：`/editor/:windowId`
- 阶段：M2/M3
- 资产：`docs/assets/frontend/effects/F01-editor-main.png`
- 页面目标：展示已加载窗口的默认编辑工作台。
- 必要区域：顶部上下文栏、左侧查看内容、中央地图、右侧编辑工具、底部状态栏。
- 主要组件：`EditorView`、`BaseMap`、`PrecipPhaseGridLayer`、`QpfEditPanel`、`PtypeEditPanel`、`OperationHistory`。
- 核心状态：`window.status=available`、`version.status=draft`、`session.status=editing`。
- 验收要点：qpf/ptype 色斑正确、地图 tooltip 正确、右侧工具可见、底部状态栏字段完整。

### F02 数据窗口选择页（已生成）

- 路由：`/editor`
- 阶段：M1/M2
- 资产：`docs/assets/frontend/effects/F02-window-selection.png`
- 页面目标：选择 case/window，查看 available/partial/invalid 状态。
- 必要区域：筛选条件（起报时间、累计长度、窗口状态）、窗口统计卡（24h/48h/168h 各状态数量）、窗口表格、窗口详情。
- 主要组件：`WindowSelector`、`WindowStatusTag`、`DataStatusPanel`。
- 核心接口：`GET /api/windows`、`POST /api/session/start`。
- 筛选说明：系统固定 0.05° 网格分辨率，不提供分辨率筛选控件（效果图中该控件为误标，后续出图时移除）。
- 验收要点：23 个窗口显示、状态 tag 正确、invalid 行不可进入编辑、选中窗口详情同步。

### F03 多边形编辑状态（已生成）

- 路由：`/editor/:windowId`
- 阶段：M2
- 资产：`docs/assets/frontend/effects/F03-polygon-editing.png`
- 页面目标：展示 polygon 选区绘制状态。
- 必要区域：地图 polygon、顶点 handle、右侧区域选择卡、选区格点数和面积。
- 主要组件：`DrawTools`、`MaskOverlayLayer`、`GridTooltip`。
- 验收要点：polygon 至少 3 点，顶点可修改，选区统计展示，未进入真实 apply。

### F04 编辑预览状态（已生成）

- 路由：`/editor/:windowId`
- 阶段：M3
- 资产：`docs/assets/frontend/effects/F04-preview-state.png`
- 页面目标：展示 preview 结果与应用前统计。
- 必要区域：预览中 badge、当前操作、选区信息、预览统计、取消/应用预览按钮。
- 主要组件：`PreviewStatsPanel`、`OperationHistory`、`MaskOverlayLayer`。
- 核心字段：`preview_id`、`op_ptype_transition`、`new_precip_needs_ptype`。
- 验收要点：preview 不写正式操作；apply 按 preview_id 应用；取消预览后恢复普通编辑状态。

### F05 版本审核页面（已生成）

- 路由：`/approval/version/:versionId`
- 阶段：M4
- 资产：`docs/assets/frontend/effects/F05-version-approval.png`
- 页面目标：审核员查看版本前后对比并审批。
- 必要区域：待审核版本列表、before/after 对比、修改摘要、审核意见、审核操作。
- 主要组件：`ApprovalView`、`VersionComparePanel`、`ApprovalActionPanel`、`ApprovalHistory`。
- 验收要点：仅 submitted 版本可审核；退回必须填写意见；通过后版本变 approved。

### F06 复盘中心页面（已生成）

- 路由：`/review/product/:reviewId`
- 阶段：M5
- 资产：`docs/assets/frontend/effects/F06-review-center.png`
- 页面目标：查看复盘图、任务状态、manifest、导出复盘包。
- 必要区域：复盘目录（左侧三级树：case → window → version）、复盘图 **2×2 网格**（before/after 上行 + delta_qpf/change_ptype 下行，或 touched_mask/changed_mask 替换下行）、右侧复盘信息、缺失字段、任务摘要、任务日志。
- 主要组件：`ReviewCenterView`、`ReviewImageGrid`（2×2 布局）、`MissingFieldsPanel`、`ReviewTaskStatus`、`ReviewExportButton`。
- 验收要点：success/partial_success/failed 均有明确视觉状态；复盘图按 2×2 网格排列；导出按钮状态正确。

## 23.4 效果图入库规则

1. 已生成图片统一放入 `docs/assets/frontend/effects/`。
2. 文件名格式固定：`Fxx-kebab-name.png`。
3. 新增图片后必须更新 `schemas/frontend_effect_screens.json` 的 `asset` 字段。
4. 效果图只作为视觉目标；组件、接口和状态以 `docs/12/17/24` 为准。
5. 若效果图与技术规格冲突，优先修订效果图或在本文件注明差异，不能让开发自行猜测。
