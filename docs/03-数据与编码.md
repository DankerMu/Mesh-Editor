## 7. 产品窗口设计

### 7.1 累计时段控制

系统只支持三种累计长度：

```text
24h
48h
168h
```

累计时段由两个参数控制：

```text
accum_hours：累计长度，取值 24 / 48 / 168
start_lead：起始预报时效
end_lead：结束预报时效，end_lead = start_lead + accum_hours
```

示例：

| 累计长度 | 起始时效 | 结束时效 | 窗口含义 |
|---:|---:|---:|---|
| 24h | +000h | +024h | 起报后 0 至 24 小时累计降水 |
| 24h | +024h | +048h | 起报后 24 至 48 小时累计降水 |
| 48h | +000h | +048h | 起报后 0 至 48 小时累计降水 |
| 48h | +048h | +096h | 起报后 48 至 96 小时累计降水 |
| 168h | +000h | +168h | 起报后 0 至 168 小时累计降水 |

### 7.2 窗口 ID 命名

建议使用统一命名规则：

```text
window_id = {init_time}_ACC{accum_hours}_{start_lead}_{end_lead}
```

示例：

```text
2026010108_ACC24_024_048
2026010108_ACC48_000_048
2026010108_ACC168_000_168
```

## 8. 数据对象设计

### 8.1 原始数据文件命名（已确认）

| 文件类型 | 命名格式 | 说明 |
|---|---|---|
| 累积降水量 | `tp_999_deeplearning_YYYYMMDDHH_hhh.txt` | 累积值，需差分得时段降水 |
| 降水概率 | `pr_999_deeplearning_YYYYMMDDHH_hhh.txt` | 0.0-1.0，辅助参考，不进入编辑 |
| 降水相态 | `ptype_999_revised_YYYYMMDDHH_hhh.txt` | 整数编码，详见第 9 节 |

数据格式：501 行 × 821 列 CSV 纯文本，逗号分隔，3h 步长，000-240h，每日 08Z/20Z 两个起报时次。

### 8.2 编辑主变量

| 变量 | 含义 | 来源 | 是否可编辑 |
|---|---|---|---:|
| qpf_before | 订正前累计降水 | tp 文件差分计算 | 否 |
| ptype_before | 订正前累计相态 | ptype 文件综合 | 否 |
| qpf_after | 订正后累计降水 | 初始化为 qpf_before | 是 |
| ptype_after | 订正后累计相态 | 初始化为 ptype_before | 是 |

### 8.2 复盘派生变量

| 变量 | 含义 |
|---|---|
| delta_qpf | qpf_after - qpf_before |
| change_ptype | ptype_before 到 ptype_after 的变化编码 |
| edit_mask | 是否被人工修改 |
| operation_mask | 单次操作影响范围 |

### 8.3 IFS 复盘变量

IFS 变量不参与交互编辑，只参与复盘绘图和归档。变量清单以后按实际测试数据确定。建议预留如下结构：

| 类别 | 变量示例 | 层次示例 |
|---|---|---|
| 高度场 | z / gh / hgt | 500 / 700 / 850 hPa |
| 风场 | u / v / wind_speed / wind_dir | 500 / 700 / 850 hPa |
| 温度 | t | 500 / 700 / 850 hPa |
| 湿度 | rh / q | 500 / 700 / 850 hPa |
| 地面或近地面 | mslp / t2m / d2m / total_precip | 按数据实际情况 |
| 其他变量 | cape / cin / pwat / freezing_level 等 | 按数据实际情况 |

## 9. 相态编码（已确认）

经分析 `reference/cdsz` 中 `GribToPng.pngCombinePtype()` 方法（GribToPng.java:250-261），**已有系统的相态编码**如下：

### 9.1 编码定义

| 编码 | 含义 | 渲染逻辑 | 数据来源 |
|---:|---|---|---|
| 0 | 无降水 | 透明（降水量 <0.1 时 alpha=0） | ptype_999_revised 文件 |
| 1 | 雨 | `getRainRgbaValue()` 绿蓝紫分级 | ptype_999_revised 文件 |
| 2 | 雪 | `getSnowRgbaValue()` 灰色分级 | ptype_999_revised 文件 |
| 3 | 雨夹雪 | `getRainSnowRgbaValue()` 固定粉紫色 | ptype_999_revised 文件 |

> **注意**：此编码与 cdsz 中 `ptypeMap`（站点天气现象：0=晴,1=雨,2=雪,3=少云,4=多云,5=阴）**不是同一套体系**。`ptypeMap` 是站点预报天气现象，包含云量类别；`ptype_999_revised` 是纯降水相态编码，仅涉及降水类型。

> **实际数据现状**：当前样本数据（5 月中旬）出现 0、1、2 三种值。编码 3（雨夹雪）在冬季数据中预期出现。

### 9.2 色标配置（已有代码定义）

#### 雨（ptype=1）：`getRainRgbaValue()`

| 量级(mm) | RGB | 颜色描述 |
|---|---|---|
| <0.1 | 透明 | — |
| 0.1 - 10 | (170, 240, 144) | 浅绿 |
| 10 - 25 | (61, 172, 5) | 深绿 |
| 25 - 50 | (104, 185, 249) | 浅蓝 |
| 50 - 100 | (5, 4, 247) | 深蓝 |
| 100 - 250 | (253, 4, 255) | 紫红 |
| ≥250 | (253, 4, 255) | 紫红 |

#### 雪（ptype=2）：`getSnowRgbaValue()`

| 量级(mm) | RGB | 颜色描述 |
|---|---|---|
| <0.1 | 透明 | — |
| 0.1 - 2.5 | (209, 210, 211) | 浅灰 |
| 2.5 - 5 | (166, 167, 168) | 中灰 |
| 5 - 10 | (117, 118, 119) | 深灰 |
| ≥10 | (77, 78, 79) | 暗灰 |

#### 雨夹雪（ptype=3）：`getRainSnowRgbaValue()`

| 量级(mm) | RGB | 颜色描述 |
|---|---|---|
| ≥0.1 | (251, 201, 252) | 固定粉紫色，不分量级 |

### 9.3 系统配置要求

本系统**沿用已有编码**，不重新定义。前端和后端的编码映射表通过配置文件管理，不硬编码：

```json
{
  "ptype_mapping": {
    "0": { "label": "无降水", "color_fn": "transparent" },
    "1": { "label": "雨", "color_fn": "rain_gradient" },
    "2": { "label": "雪", "color_fn": "snow_gradient" },
    "3": { "label": "雨夹雪", "color_fn": "rain_snow_fixed" }
  }
}
```
