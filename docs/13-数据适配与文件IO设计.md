# 13. 数据适配与文件 IO 设计

## 13.1 设计目标

本文件定义原始数据读取、累计窗口构建、相态合成、QC mask、缓存、归档路径和文件格式，保证数据接入、编辑、复盘、发布使用同一套文件口径。

---

## 13.2 数据源

| 数据 | 文件模式 | 用途 |
|---|---|---|
| tp | `tp_999_deeplearning_YYYYMMDDHH_hhh.txt` | 从起报起累计降水，差分得到 qpf_window |
| pr | `pr_999_deeplearning_YYYYMMDDHH_hhh.txt` | 降水概率，暂不编辑，可保留为复盘或后续辅助 |
| ptype | `ptype_999_revised_YYYYMMDDHH_hhh.txt` | 3h 相态，合成累计窗口 ptype_before |
| IFS | GRIB/GDS/其他 | 复盘绘图变量，不进入编辑主流程 |

固定网格：

```text
ny = 501
nx = 821
lon_min = 70
lon_max = 111
lat_min = 25
lat_max = 50
dlon = 0.05
dlat = 0.05
projection = EPSG:4326
row_col_order = y_desc_x_asc
```

## 13.3 基础读取函数

```python
def read_txt_grid(path: Path, dtype: str, expected_shape=(501, 821)) -> GridReadResult:
    """读取逗号分隔 txt 网格文件。"""
```

返回：

```python
@dataclass
class GridReadResult:
    array: np.ndarray
    path: Path
    shape: tuple[int, int]
    dtype: str
    nan_count: int
    invalid_count: int
```

校验规则：

| 检查 | 处理 |
|---|---|
| 文件不存在 | 抛 `FILE_NOT_FOUND` |
| shape 不为 501×821 | 抛 `GRID_SHAPE_MISMATCH` |
| 存在 NaN | 记录 nan_count，生成 missing_mask |
| ptype 非 0/1/2/3 | 标记 invalid_count，窗口 qc_status=warn 或 fail |

## 13.4 qpf_window 构建

函数：

```python
def build_qpf_window(case_id: str, start_lead: int, end_lead: int, config: ProductConfig) -> QpfBuildResult:
    ...
```

规则：

```text
qpf_before = tp(end_lead) - tp(start_lead)
```

`start_lead=000` 优先级：

```text
1. 如果 tp_000 文件存在，读取实际文件；
2. 如果不存在且 allow_zero_start_lead_fallback=true，使用全零数组；
3. 否则窗口 invalid。
```

返回：

```python
@dataclass
class QpfBuildResult:
    qpf: np.ndarray
    start_tp_path: Path | None
    end_tp_path: Path
    negative_mask: np.ndarray
    missing_mask: np.ndarray
    negative_count: int
    negative_min_value: float | None
    negative_abs_max: float | None
    missing_count: int
    qc_status: Literal['pass', 'warn', 'fail']
```

负值处理：

```text
- 负值格点不参与编辑和统计。
- qpf_before 原始值可保留在 raw_qpf_before，用于数据诊断。
- 渲染和编辑使用 masked qpf，其中负值格点透明或斜线标注。
```

建议保存：

```text
original/qpf_before.npz
original/qpf_raw_before.npz，可选
original/negative_qpf_mask.npz
original/missing_mask.npz
```

## 13.5 ptype_window 构建

函数：

```python
def build_ptype_window(case_id: str, start_lead: int, end_lead: int, config: ProductConfig) -> PtypeBuildResult:
    ...
```

时效范围：

```text
(start_lead, end_lead]
```

示例：

```text
024-048h → 027,030,033,036,039,042,045,048
```

步骤：

1. 读取每个 3h ptype 文件；缺失则记录到 `missing_leads`。
2. 对每个 3h 时效计算：

```text
qpf_step[t] = tp[t] - tp[t-3]
```

3. 仅 `qpf_step[t] > qpf_threshold` 的时效参与相态统计。
4. 逐格点设置：

```text
ptype=1 → has_rain
ptype=2 → has_snow
ptype=3 → has_rain + has_snow
```

5. 合成：

| has_rain | has_snow | ptype_before |
|---:|---:|---:|
| 0 | 0 | 0 |
| 1 | 0 | 1 |
| 0 | 1 | 2 |
| 1 | 1 | 3 |

返回：

```python
@dataclass
class PtypeBuildResult:
    ptype: np.ndarray
    valid_leads: list[int]
    missing_leads: list[int]
    missing_count: int
    qc_status: Literal['pass', 'warn', 'fail']
```

状态建议：

```text
全部 ptype 缺失 → product_window invalid
部分 ptype 缺失 → product_window partial，qc_status=warn
无缺失 → available/pass
```

## 13.9 PathBuilder 规则

路径必须由 `PathBuilder` 统一生成。PathBuilder 输入 `case_id/window_id/session_id/version_id/review_id`，输出 pathlib.Path。

路径生成不应依赖前端传入路径，避免路径穿越和归档混乱。

## 13.10 缓存策略

| 缓存 | 位置 | TTL | 清理 |
|---|---|---:|---|
| preview | `tmp/previews/{session_id}/{preview_id}.npz` | 10 分钟 | apply/session end/定时清理 |
| session_state | `archive/.../sessions/{session_id}` | session 有效期 | saved/discarded 后可保留或压缩 |
| plot intermediate | `tmp/plots/{review_id}` | 任务完成后 1 天 | 定时清理 |
| scan temp | `tmp/scans/{scan_id}` | 扫描完成后立即清理 | 自动 |

---

## 13.11 文件写入原子性

所有正式归档文件采用：

```text
write tmp → fsync → rename
```

避免服务中断时出现半写入文件。

伪代码：

```python
def atomic_write_npz(path: Path, **arrays):
    tmp = path.with_suffix(path.suffix + '.tmp')
    np.savez_compressed(tmp, **arrays)
    os.replace(tmp, path)
```
