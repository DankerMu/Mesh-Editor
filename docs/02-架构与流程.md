## 5. 系统总体架构

### 5.1 技术选型（已确认）

| 层级 | 选型 | 说明 |
|---|---|---|
| 前端框架 | Vue 3 + TypeScript | 参考 yy-webgis-bj 项目 |
| 地图引擎 | OpenLayers 10.x | 支持 EPSG:4326 等经纬度投影，已有 WebGL 网格瓦片渲染参考 |
| UI 组件库 | TDesign Vue Next | 参考项目已集成 |
| 后端框架 | Python FastAPI | 异步、高性能，与 NumPy/xarray 生态无缝对接 |
| 格点处理 | NumPy / xarray / shapely | 网格编辑、mask 生成、差异计算 |
| 绘图服务 | Matplotlib + Cartopy | 复用已有绘图脚本（PlateCarree 投影） |
| 数据库 | SQLite（原型）→ PostgreSQL（生产） | 轻量起步 |
| 数据格式 | NPZ + PNG + JSONL + CSV（原型）→ NetCDF（生产） | 渐进升级 |
| 部署方式 | Docker | 开发期本地运行本地测试，最终打镜像部署 |
| 投影坐标系 | EPSG:4326（等经纬度） | 前后端、绘图代码统一 |

#### 输入数据

```text
├── 固定区域底图 PNG
├── 固定区域 0.05° 等经纬度网格定义（501×821，EPSG:4326）
├── 累积降水量 tp：3h 步长，000-240h，差分得时段累计
├── 降水概率 pr：辅助参考，不进入编辑
├── 累计相态场 ptype
└── EC IFS 复盘诊断场（GRIB 格式）
    ├── 500hPa / 700hPa / 850hPa 高度场
    ├── 500hPa / 700hPa / 850hPa 风场
    ├── 500hPa / 700hPa / 850hPa 温度场
    ├── 500hPa / 700hPa / 850hPa 湿度场
    └── 其他变量（CAPE、PRMSL、TCWV 等）
```

#### 部署（Docker）

```text
├── 前端：Nginx 容器托管 Vue 构建产物
├── 后端：FastAPI 容器（含 NumPy/Matplotlib 环境）
└── 数据卷：挂载格点数据、绘图产物、归档目录
```

## 6. 业务流程

### 6.1 编辑流程

```text
1. 用户选择起报时间
2. 用户选择累计长度：24h / 48h / 168h
3. 用户选择起始时效
4. 系统自动计算结束时效
5. 系统加载 qpf_before、ptype_before
6. 系统初始化 qpf_after = qpf_before，ptype_after = ptype_before
7. 用户选择多边形、绘制笔或笔刷
8. 用户选择编辑动作：降水定值、增加、减少、乘系数、清零，或相态改雨/雨夹雪/雪/无降水
9. 系统生成预览
10. 用户确认应用
11. 系统记录操作日志
12. 用户保存版本
13. 系统生成差异场、相态变化、修改掩膜
14. 系统调用基础绘图代码
15. 系统调用复盘绘图代码
16. 系统保存复盘数据包
```

### 6.2 复盘流程

```text
1. 接收已保存的 edit_version
2. 定位当前累计时段
3. 计算前 24h 与后 24h 窗口
4. 读取当前窗口 qpf/ptype 订正前后数据
5. 读取前后 24h 的 qpf/ptype 数据
6. 读取对应 IFS 复盘变量
7. 生成 plot_input_manifest.json
8. 调用复盘绘图代码
9. 保存 review_composite.png 及其他复盘图
10. 记录复盘产品信息
```
