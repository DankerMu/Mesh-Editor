## 18. 技术路线（已确认）

### 18.1 确定架构

```text
前端：Vue 3 + OpenLayers 10.x + TDesign Vue Next + TypeScript
后端：Python FastAPI
格点处理：NumPy / xarray / shapely
绘图服务：Matplotlib + Cartopy（复用已有绘图脚本）
数据格式：NPZ + PNG + JSONL + CSV（原型）→ NetCDF（生产归档）
数据库：SQLite（原型）→ PostgreSQL（生产）
文件存储：本地文件系统，后续可扩展 MinIO
投影坐标系：EPSG:4326（等经纬度投影），前后端、绘图代码统一
部署：Docker（前端 Nginx 容器 + 后端 FastAPI 容器 + 数据卷）
开发流程：本地开发本地测试 → 最终打镜像部署
```

### 18.2 选型依据

| 选型 | 依据 |
|---|---|
| Vue 3 + OL | 参考项目 yy-webgis-bj 已验证，含 WebGL 网格瓦片、投影切换、时间轴等可复用组件 |
| TDesign | 参考项目已集成，组件丰富，适合后台管理类界面 |
| FastAPI | 异步高性能，与 NumPy/Matplotlib 生态无缝对接 |
| EPSG:4326 | 已有绘图代码全部使用 PlateCarree，网格数据本身为等经纬度，无需投影转换 |
| Docker | 隔离运行环境（尤其 Matplotlib 字体/Cartopy 地图数据），便于在生产内网部署 |

### 18.3 不建议路线

不建议按 GFIE 的老式重客户端方式直接复刻。原因是：

1. 本项目区域固定、底图固定，Web/轻量前端即可满足。
2. 本项目核心是数据留痕和复盘归档，Web 后端更易做版本、日志、批量任务。
3. 绘图代码已有且为 Python，FastAPI 直接调用最简单。

## 19. 绘图代码接入方案

### 19.1 已有绘图脚本清单

经对 reference/要素场绘图/ 目录完整分析，已有以下绘图能力：

| 脚本 | 功能 | 要素 | 地理范围 |
|---|---|---|---|
| circulation.py | 高空环流形势 | 相对湿度填色 + 位势高度等值线 + 风场 | 70-140E, 15-55N |
| rain.py / rain24.py | 累积降水 | APCP 差分，6 级色标 | 70-115E, 25-50N |
| element.py | 多要素场 | CAPE/TMP_HIGH/SPFH/TCWV/变温/强风 | 70-115E, 25-50N |
| surface.py | 地面形势 | 海平面气压填色 + 10m 风场 | 70-140E, 15-55N |
| cape.py | 对流有效位能 | CAPE 单要素 | 70-115E, 25-50N |
| tmp_high.py | 地面高温 | 2m 温度 | 70-115E, 25-50N |
| poumian1.py | 站点剖面 | 温度/湿度/垂直速度/风/降水 | 57 个站点 |
| app1.py | Flask 后端 | 图片托管服务 | 监听 192.168.2.80:8080 |
| data_utils.py | 数据工具库 | 地图边界读取、GRIB 解析、时间转换 | — |

所有绘图脚本统一使用：
- **投影**：PlateCarree（等经纬度）
- **数据读取**：meteva.base（GDS 格式）和 xarray（GRIB 格式）
- **边界数据**：GeoJSON 国界/省界
- **时间转换**：UTC → Asia/Shanghai (+8h)
- **区域划分**：All / NW / SW / East 四个子区域

### 19.2 接入策略

已有绘图脚本以"直接生成 PNG 文件"为目标，适合通过 FastAPI 后台任务异步调用。接入分两步：

**第一步：基础出图（降水/相态编辑前后对比）**

需要新建适配函数，读取编辑前后的 NumPy 数组，复用 rain.py 的色标和绘图逻辑：

```python
def plot_product(
    data: np.ndarray,
    grid_def: dict,
    product_type: str,
    metadata: dict,
    output_path: str
) -> str:
    ...
```

生成产物包括：before_product.png、after_product.png、delta_qpf.png、change_ptype.png、edit_mask.png。

**第二步：复盘组合图**

直接调用已有脚本生成环流、降水、要素场、地面场、剖面图，通过 manifest 组织输入：

```python
def plot_review_composite(
    plot_input_manifest_path: str,
    output_path: str,
    config_path: str | None = None
) -> str:
    ...
```
