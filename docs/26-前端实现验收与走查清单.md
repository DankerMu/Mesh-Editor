# 26. 前端实现验收与走查清单

## 26.1 文档定位

本文用于前端页面实现后的验收、走查和截图回归。验收目标不是像素级一模一样，而是保证结构、组件、状态、色彩、交互、文案和业务流与效果图一致。

---

## 26.2 通用验收标准

每个页面必须满足：

```text
1. 使用统一 AppShell、顶部导航、中文文案。
2. 页面中的状态 tag 使用统一 token。
3. loading / empty / error / disabled 状态完整。
4. 权限按钮符合角色权限规则。
5. API 错误显示 trace_id。
6. 主要按钮启禁用符合 docs/24。
7. 页面在 1440×900 下无明显溢出。
8. CI 中单元测试和 E2E 测试通过。
```

## 26.5 视觉回归建议

使用 Playwright 截图回归：

```text
- 每个效果图页面至少一个 screenshot test。
- 默认视口：1672×941 或 1440×900。
- 允许少量地图栅格差异，但布局、组件、文案、状态 tag 必须一致。
```

建议截图测试路由：

```text

/editor?mock=windows
/editor/ACC24_024_048?mock=loaded
```

## 26.7 不通过判定

以下情况视为 UI/UX 验收不通过：

```text
- 菜单命名或顺序与规范不一致。
- 状态颜色与 token 不一致。
- 按钮启禁用规则错误。
- 地图 tooltip 字段缺失。
- partial/failed 状态没有明显提示。
- 审核/发布/导出等关键操作无确认或反馈。
- 错误提示没有 trace_id。
- 字段二进制加载失败后仍渲染错误数据。
```
