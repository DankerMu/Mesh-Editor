# 11. 后端工程设计

## 11.1 设计目标

后端采用 Python FastAPI。本文定义工程目录、服务边界、核心类、存储路径和模块依赖，避免进入开发后出现大而乱的接口文件、重复路径拼接、跨服务直接改状态等问题。

## 11.3 分层职责

| 层 | 职责 | 禁止事项 |
|---|---|---|
| api/routes | 接收请求、鉴权、调用服务、返回 Schema | 不写业务算法、不直接读写文件 |
| services | 编排业务流程、状态机、事务 | 不做底层数值算法，不手写路径 |
| engines | 数组计算、mask、差分、ptype 合成、统计 | 不直接写数据库状态 |
| repositories | 数据库 CRUD | 不包含业务判断 |
| storage | 路径生成、文件读写、临时缓存、归档 | 不判断业务状态 |
| plotters | 绘图纯函数 | 不读数据库、不判断任务状态 |
| workers | 异步任务执行 | 不绕过 Service 直接改核心业务状态 |

## 11.4 核心服务设计

### 11.4.1 DataScanService

职责：

```text
- 根据 case_id 定位数据目录
- 扫描 tp/ptype/pr 文件
- 创建或更新 ForecastCase
- 枚举 ProductWindow
- 调用 QpfBuilder 构建 qpf_before
- 调用 PtypeBuilder 构建 ptype_before
- 生成 v000 零场派生文件（delta_qpf / change_ptype / touched_mask / changed_mask 全零 501×821）
- 插入 edit_version v000_original 记录
- 执行 QC 并更新 product_window.status/qc_status
- 写 data_scan_log
```

主要方法：

```python
class DataScanService:
    def scan_case(self, case_id: str, scan_type: str, user_id: str | None) -> DataScanResult: ...
    def build_window(self, case_id: str, accum_hours: int, start_lead: int) -> WindowBuildResult: ...
```

边界：不负责编辑、审核、绘图。

### 11.4.2 WindowService

职责：

```text
- 查询窗口列表
- 查询窗口数据摘要
- 判断窗口是否可编辑
- 读取 latest released/draft/version 信息
```

主要方法：

```python
class WindowService:
    def list_windows(self, case_id: str, accum_hours: int | None, status: str | None) -> list[ProductWindowDTO]: ...
    def assert_window_editable(self, window_id: str) -> ProductWindow: ...
```

### 11.4.3 EditSessionService

职责：

```text
- 创建编辑 session
- 加载 session 当前数组状态
- 撤销/重做
- 放弃 session
- session 过期清理
```

主要方法：

```python
class EditSessionService:
    def start_session(self, window_id: str, user_id: str) -> EditSessionDTO: ...
    def load_session_state(self, session_id: str) -> SessionStateDTO: ...
    def undo(self, session_id: str) -> UndoRedoResult: ...
    def redo(self, session_id: str) -> UndoRedoResult: ...
    def discard(self, session_id: str) -> None: ...
```

### 11.4.4 EditEngineService

职责：

```text
- 基于 session 当前数组生成 preview
- 缓存 preview
- apply preview
- 生成 edit_operation
- 计算 before/after stats
```

主要方法：

```python
class EditEngineService:
    def preview(self, req: EditPreviewRequest) -> EditPreviewResponse: ...
    def apply(self, session_id: str, preview_id: str) -> EditApplyResponse: ...
```

约束：preview 不修改 session 状态；apply 不重新计算 preview。

### 11.4.5 VersionService

职责：

```text
- 保存版本
- 生成 delta_qpf / change_ptype / touched_mask / changed_mask
- 写 edit_version
- 关联 session
- 触发基础出图和可选复盘任务
- 提交审核
```

主要方法：

```python
class VersionService:
    def save_version(self, session_id: str, generate_review: bool) -> VersionSaveResult: ...
    def submit(self, version_id: str, user_id: str) -> EditVersion: ...
```

### 11.4.6 ApprovalService / ReleaseService

职责：

```text
ApprovalService：审核通过/退回，写 review_approval。
ReleaseService：发布 approved 版本，supersede 旧发布，按产物契约（docs/13 §13.8.1）构建发布目录并生成 product_manifest.json（含 npz→txt 转换）。
```

主要方法：

```python
class ApprovalService:
    def review(self, version_id: str, reviewer_id: str, action: str, comment: str | None) -> ApprovalResult: ...

class ReleaseService:
    def release(self, version_id: str, released_by: str) -> ReleaseResult: ...
```

### 11.4.7 ReviewService / PlotTaskService

职责：

```text
ReviewService：组装 review_payload，创建 review_product，提交绘图任务。
PlotTaskService：队列调度、状态更新、重试、失败日志。
```

主要方法：

```python
class ReviewService:
    def generate_review(self, window_id: str, version_id: str, template_id: str) -> ReviewProduct: ...
    def regenerate(self, review_id: str) -> ReviewProduct: ...
    def export_package(self, window_id: str, version_id: str) -> Path: ...

class PlotTaskService:
    def enqueue(self, review_id: str) -> None: ...
    def run_next(self) -> None: ...
    def retry(self, review_id: str) -> None: ...
```

### 11.4.8 AuthService / AuditService / ConfigService / StatsService

职责：

```text
AuthService：登录、密码校验、JWT。
AuditService：记录关键动作。
ConfigService：配置校验、配置快照、配置读取。
StatsService：操作统计、相态转换统计、导出 CSV。
```

## 11.5 Engine 设计

| Engine | 输入 | 输出 | 说明 |
|---|---|---|---|
| GridIO | 文件路径 | np.ndarray + 元数据 | 读取 501×821 CSV、NPZ、NetCDF |
| QpfBuilder | case_id, start_lead, end_lead | qpf_before, QC mask | tp 差分 |
| PtypeBuilder | case_id, start_lead, end_lead | ptype_before | 窗口相态合成 |
| MaskBuilder | geometry, grid_def | bool mask | polygon/line_buffer/brush_path 转栅格 |
| EditOps | qpf/ptype/mask/params | EditResult | 数值修改 |
| Consistency | qpf/ptype | fixed arrays + warnings | qpf/ptype 一致性 |
| Area | mask, grid_def | area_km2 | 近似或球面面积 |
| StatsCalc | before/after/mask | stats | 统计和变化矩阵 |

## 11.6 Repository 设计

Repository 只做数据库操作，禁止写业务状态机逻辑。

示例：

```python
class EditVersionRepository:
    def get(self, version_id: str) -> EditVersion: ...
    def get_latest_for_window(self, window_id: str) -> EditVersion | None: ...
    def create(self, model: EditVersionCreate) -> EditVersion: ...
    def update_status(self, version_id: str, status: str) -> None: ...
```

状态合法性由 Service 校验。

## 11.7 PathBuilder 统一路径

所有路径通过 `storage/path_builder.py` 生成。

```python
class PathBuilder:
    def case_root(self, case_id: str) -> Path: ...
    def window_root(self, window_id: str) -> Path: ...
    def original_field(self, window_id: str, name: str) -> Path: ...
    def session_root(self, session_id: str) -> Path: ...
    def preview_file(self, session_id: str, preview_id: str) -> Path: ...
    def version_root(self, window_id: str, version_id: str) -> Path: ...
    def review_root(self, window_id: str, review_id: str) -> Path: ...
    def release_root(self, window_id: str, version_id: str) -> Path: ...
```

禁止在业务代码中手写：

```python
Path('archive') / case_id / ...
```

## 11.8 事务边界

| 操作 | 事务范围 |
|---|---|
| 数据扫描单个窗口 | 构建文件成功后再写 product_window；失败写 data_scan_log |
| apply preview | 写 session 状态文件、mask 文件、edit_operation 应保持一致 |
| 保存版本 | 写 version 文件、edit_version、session 状态应作为一个业务事务 |
| 审核 | edit_version 状态 + review_approval 同事务 |
| 发布 | supersede 旧 release + 当前 release + edit_version 状态 同事务 |
| 绘图任务 | review_product 状态变化可分阶段提交 |

文件写入无法完全回滚，需采用临时文件写入后原子 rename：

```text
write xxx.tmp → fsync → rename xxx.tmp to xxx.npz
```

## 11.10 并发策略

策略：同一 `window_id` 同时只允许一个 editing session（窗口级排他锁）。

```json
{
  "edit_lock_policy": "single_editing_session_per_window",
  "session_ttl_minutes": 240
}
```

`/api/session/start` 检查：若同一 window 存在 `status=editing` 且未过期的 session，返回 `WINDOW_LOCKED`。

session 超时（TTL 到期）后自动释放锁，允许其他用户开始新 session。

## 11.10.1 数据库索引

```sql
-- 窗口查询
CREATE INDEX idx_product_window_case ON product_window(case_id);
CREATE INDEX idx_product_window_status ON product_window(case_id, status, qc_status);

-- 版本查询
CREATE INDEX idx_edit_version_window ON edit_version(window_id, version_no);

-- Session 锁检查
CREATE INDEX idx_edit_session_window_status ON edit_session(window_id, status);

-- 操作重放
CREATE INDEX idx_edit_operation_session_seq ON edit_operation(session_id, sequence_no);

-- 复盘查询
CREATE INDEX idx_review_product_version ON review_product(version_id);

-- 审计日志查询
CREATE INDEX idx_audit_log_user_time ON audit_log(user_id, created_at);
CREATE INDEX idx_audit_log_resource ON audit_log(resource_type, resource_id);

-- 发布唯一性（同一 window 同时只能有一个 active release）
-- PostgreSQL：
CREATE UNIQUE INDEX ux_release_active_window ON release_product(window_id) WHERE release_status = 'active';
-- SQLite 不支持 partial index，ReleaseService 事务内检查唯一性
```

### 11.10.2 迁移策略

原型期使用 SQLite + Alembic：

```text
1. alembic init 生成迁移目录
2. 首个迁移 v001_initial_schema 包含全部表和索引
3. 后续每个 schema 变更对应一个迁移版本
4. CI 中执行 alembic upgrade head 验证迁移链完整性
```

生产迁移到 PostgreSQL 时：
- partial index（`ux_release_active_window`）可直接使用
- TEXT 类型无需变更（PostgreSQL TEXT 性能等同 VARCHAR）
- DATETIME 字段从 TEXT 改为 TIMESTAMPTZ，迁移脚本处理
