# 12. 前端工程设计

## 12.1 设计目标

前端采用 Vue 3 + TypeScript + OpenLayers + TDesign。本文将现有”界面布局”扩展为可开发的路由、组件、Store、地图图层、权限与错误处理设计。

前端视觉 token、布局尺寸、效果图索引、组件交互状态、Mock 数据和验收走查详见 docs/22-26。每个页面开发前必须查阅 docs/23 中对应效果图规格；组件状态和按钮启禁用以 docs/24 为准；Mock 数据和二进制字段加载以 docs/25 为准；验收以 docs/26 为准。

### 12.1.2 前端技术栈冻结

```text
- 构建工具：Vite
- 框架：Vue 3 + TypeScript
- 路由：Vue Router 4
- 状态管理：Pinia
- UI 组件：TDesign Vue Next
- 地图：OpenLayers 10.x
- 图表：ECharts
- HTTP：axios
- 单元测试：Vitest + Vue Test Utils
- E2E：Playwright
- 代码规范：ESLint + Prettier + TypeScript strict
```

### 12.1.1 中文化约束

本产品面向中国气象业务用户，全界面中文，不做国际化：

```text
- TDesign 配置 locale 为 zh-CN
- 所有 UI 文案（菜单、按钮、表头、提示、对话框）使用中文
- 错误提示使用中文（前端维护 error_codes → 中文 message 映射表，从 schemas/error_codes.json 生成）
- 日期时间显示格式：YYYY-MM-DD HH:mm（北京时间 UTC+8）
- 数值格式：千分位不加逗号，小数点用点号
- 地图标注、图例、色标标签使用中文
```

## 12.5 地图组件设计

### 12.5.1 BaseMap.vue

职责：初始化 OpenLayers map，注册 EPSG:4326，加载固定底图、编辑图层和交互工具。

属性：

```ts
props: {
  gridDef: GridDefinition
  baseMapUrl: string
}
```

事件：

```ts
emit('grid-hover', GridHoverPayload)
emit('mask-created', MaskGeometry)
emit('map-ready', Map)
```

### 12.5.2 PrecipPhaseGridLayer.ts

输入：

```ts
qpfArray: Float32Array
ptypeArray: Uint8Array
gridDef: GridDefinition
colorConfig: ProductColorConfig
touchedMask?: Uint8Array
changedMask?: Uint8Array
invalidMask?: Uint8Array
```

职责：

```text
- 使用 DataTileSource + WebGLTileLayer 渲染 qpf+ptype 双变量色斑
- 鼠标拾取返回 qpf_before/qpf_after/ptype_before/ptype_after/is_edited
- 支持 updateData(qpf, ptype)
- 支持显示 invalid mask 斜线纹理或半透明覆盖
```

关键约束：

```text
1. 不使用全球 wrapX。
2. 经纬度超出 grid extent 返回 null。
3. ptype=0 或 qpf<0.1 渲染透明。
4. ptype=1 用雨色标，ptype=2 用雪色标，ptype=3 用雨夹雪固定色。
```

### 12.5.3 DrawTools.vue

支持：

```text
polygon
line_buffer
brush_path
clear_selection
```

输出统一 `MaskGeometry`：

```ts
interface PolygonGeometry {
  type: 'polygon'
  coordinates: [number, number][]
}

interface LineBufferGeometry {
  type: 'line_buffer'
  widthGrid: number
  coordinates: [number, number][]
}

interface BrushPathGeometry {
  type: 'brush_path'
  radiusGrid: number
  points: [number, number][]
}
```

## 12.7 API 调用约束

1. 所有请求通过 `api/http.ts`，自动注入 JWT。
2. 统一处理 `code/message/data/trace_id` 响应结构。
3. 403 显示权限不足；409 显示状态冲突；5xx 显示错误面板和 trace_id。
4. 文件下载统一通过 `DownloadButton`。
5. 绘图任务和数据扫描任务使用轮询或后续 WebSocket 扩展。

### 12.11.3 查看模式与图层切换规则

左侧查看模式为单选，切换时地图显示对应图层：

| 模式 | 显示图层 | 说明 |
|---|---|---|
| before | qpf_before + ptype_before | 原始预报 |
| after（默认） | qpf_after + ptype_after | 当前订正结果 |
| delta | delta_qpf 色标 | qpf_after − qpf_before，正值暖色、负值冷色 |
| change | change_ptype 色标 | ptype 变化：不变=透明，0→1/2/3、1→2/3 等用分类色 |
| touched | touched_mask 半透明覆盖 | 所有曾被编辑过的格点 |
| changed | changed_mask 半透明覆盖 | 值实际发生变化的格点（含浮点容差） |
| review | after + touched_mask 边框叠加 | 审核视角：订正结果 + 修改范围 |

附加规则：

```text
- 默认进入 after 模式
- preview 期间：在当前模式图层上叠加 preview result 半透明覆盖，显示"预览中"标识
- preview overlay 使用虚线边框区分已应用操作和待确认预览
- 切换模式不影响当前选区和编辑参数
- before/after 不支持分屏（简化 M2 实现，后续版本可扩展）
- 不提供透明度滑块（色斑透明度固定 0.7）
```

### 12.11.4 按钮启禁用规则

```text
┌─────────────────────┬─────────────────────────────────────────┐
│ 按钮                │ 禁用条件                                 │
├─────────────────────┼─────────────────────────────────────────┤
│ 预览 (Preview)      │ sessionId == null || mask == null        │
│ 应用 (Apply)        │ previewId == null                       │
│ 保存 (Save)         │ dirty == false || saveLoading == true    │
│ 撤销 (Undo)         │ canUndo == false                        │
│ 重做 (Redo)         │ canRedo == false                        │
│ 提交审核 (Submit)   │ version.status != 'draft'               │
│ 选区工具            │ sessionId == null                       │
│ 降水/相态调整面板    │ mask == null                            │
└─────────────────────┴─────────────────────────────────────────┘

加载状态下按钮行为：
- previewLoading == true → 预览按钮显示 loading spinner，禁止重复点击
- applyLoading == true → 应用按钮显示 loading spinner
- saveLoading == true → 保存按钮显示 loading spinner
- loadingFields == true → 地图显示全屏 loading 遮罩
- loadingSession == true → 顶部显示"正在加载会话..."
```

### 12.11.5 新增降水相态选择弹窗

当 preview 返回 `new_precip_needs_ptype = true` 时触发：

```text
触发条件：
  编辑操作使原本 qpf < 0.1 的格点变为 qpf >= 0.1（新增降水落区）
  这些格点需要指定相态

弹窗流程：
  1. preview 返回 → 检查 response.new_precip_needs_ptype
  2. 若为 true → 弹出 TDesign Dialog
  3. 弹窗标题："新增降水落区相态选择"
  4. 弹窗内容：
     - 提示文字："本次编辑在 {count} 个格点新增了降水，请选择这些格点的降水相态"
     - 单选组：雨 (ptype=1) / 雪 (ptype=2) / 雨夹雪 (ptype=3)
     - 默认选中：雨
  5. 确认 → apply 请求携带 target_ptype 参数
  6. 取消 → 不执行 apply，保持 preview 状态

弹窗组件：NewPrecipPtypeDialog.vue
  props: { visible, affectedCount }
  emit: ('confirm', targetPtype) | ('cancel')
```

### 12.11.6 选区工具交互细节

#### polygon（多边形）

```text
- 左键单击：添加顶点
- 双击：完成多边形闭合
- Esc：取消当前绘制
- Backspace/Delete：删除上一个顶点
- 右键：取消当前绘制
- 最少 3 个顶点才可完成
- 完成后高亮显示选区，可重新绘制覆盖
```

#### line_buffer（线缓冲区）

```text
- 左键单击：添加线段节点
- 双击：完成线段
- 右侧工具面板设置 width_grid（缓冲宽度，单位：格点数）
- width_grid 范围：1-50，默认 5
- 完成后显示线段 + 两侧缓冲区半透明填充
- Esc：取消当前绘制
- Backspace/Delete：删除上一个节点
```

#### brush_path（笔刷路径）

```text
- 按住左键拖动：绘制路径
- 松开左键：完成一次 brush geometry
- 鼠标滚轮：调整 radius_grid（笔刷半径，单位：格点数）
- 快捷键 [ / ]：减小/增大 radius_grid
- radius_grid 范围：1-30，默认 3
- 绘制时实时显示笔刷覆盖范围（圆形光标）
- 多次绘制合并为一个 brush_path geometry
- Esc：清除当前未提交的笔刷路径
```

#### clear_selection（清除选区）

```text
- 点击清除按钮或快捷键 Ctrl+Shift+A：清除当前所有选区
- 清除后 currentMaskGeometry 置为 null
- 降水/相态调整面板恢复禁用状态
```
