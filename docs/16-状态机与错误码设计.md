# 16. 状态机与错误码设计

## 16.1 设计目标

状态机和错误码是后端服务、前端展示、测试用例对齐的核心。所有状态变化必须通过 Service 层完成，禁止直接在 API 或 Repository 中随意更新状态。

---

## 16.2 ProductWindow 状态机

```text
pending → available
pending → partial
pending → invalid
available → archived
partial → available，重新扫描补齐数据
partial → invalid，关键数据确认不可用
invalid → available，重新扫描修复
```

| 状态 | 含义 | 可编辑 | 可复盘 | 可发布 |
|---|---|:---:|:---:|:---:|
| pending | 等待数据 | 否 | 否 | 否 |
| available | 数据完整 | 是 | 是 | 是 |
| partial | 部分数据缺失但可降级 | 可配置 | 是 | 需审核确认 |
| invalid | 关键数据缺失或 QC fail | 否 | 否 | 否 |
| archived | 历史归档 | 否 | 是 | 否 |

关键规则：

```text
status=invalid 时 /api/session/start 必须返回 WINDOW_NOT_EDITABLE。
status=partial 是否可编辑由 product_config.allow_partial_window_edit 控制。
```

## 16.3 EditSession 状态机

```text
editing → saved
editing → discarded
editing → expired
```

| 状态 | 含义 | 可 preview/apply | 可 save |
|---|---|:---:|:---:|
| editing | 编辑中 | 是 | 是 |
| saved | 已保存成版本 | 否 | 否 |
| discarded | 已放弃 | 否 | 否 |
| expired | 超时过期 | 否 | 否 |

建议配置：

```json
{"session_ttl_minutes": 240}
```

---

## 16.4 Preview 状态机

```text
created → applied
created → expired
created → overwritten
created → discarded
```

| 状态 | 含义 |
|---|---|
| created | 已生成，可 apply |
| applied | 已应用，缓存删除 |
| expired | TTL 过期 |
| overwritten | 同 session 新 preview 覆盖旧 preview |
| discarded | session 结束清理 |

API 规则：

```text
apply 只能作用于 created 状态 preview。
expired/overwritten/applied 均返回 PREVIEW_EXPIRED 或 PREVIEW_CONFLICT。
```

## 16.5 EditVersion 状态机

```text
draft → submitted → approved → released → superseded
submitted → rejected
rejected → 新 session → 新 draft
```

| 状态 | 含义 | 可提交 | 可审核 | 可发布 | 可作为编辑基线 |
|---|---|:---:|:---:|:---:|:---:|
| draft | 保存草稿 | 是 | 否 | 否 | 是 |
| submitted | 待审核 | 否 | 是 | 否 | 否 |
| approved | 审核通过 | 否 | 否 | 是 | 是 |
| rejected | 审核退回 | 否 | 否 | 否 | 是 |
| released | 当前或曾经发布 | 否 | 否 | 否 | 是 |
| superseded | 被新发布替代 | 否 | 否 | 否 | 是 |

规则：

```text
1. draft 只能 submit。
2. submitted 只能 approve/reject。
3. approved 只能 release。
4. released 遇到同窗口新 release 时变 superseded。
5. rejected 不可直接 approve，必须基于它新建 session 修订。
```

## 16.6 ReleaseProduct 状态机

```text
active → superseded
```

规则：

```text
同一 window_id 同时只能有一个 active release_product。
发布新版本时旧 active 自动 superseded。
```

## 16.7 PlotTask / ReviewProduct 状态机

```text
pending → running → success
pending → running → partial_success
pending → running → failed → pending(retry)
failed → permanently_failed
```

| 状态 | 含义 | 可重试 |
|---|---|:---:|
| pending | 排队中 | 否 |
| running | 运行中 | 否 |
| success | 完全成功 | 可重新生成 |
| partial_success | 部分成功 | 可重新生成 |
| failed | 失败但未超过重试次数 | 是 |
| permanently_failed | 超过最大重试次数 | 管理员可手动重试 |

## 16.8 并发与锁

推荐策略：允许多 session，但保存时检查基线版本。

```text
if session.base_version_id != latest_version_id:
    return VERSION_BASE_OUTDATED
```

可选策略：同 window 单编辑锁。

```text
/api/session/start:
  若存在 status=editing 且未过期 session，则返回 WINDOW_LOCKED
```

配置：

```json
{
  "edit_lock_policy": "multi_session_with_base_check",
  "session_ttl_minutes": 240
}
```

## 16.10 错误码总表

### 认证与权限

| code | HTTP | 说明 |
|---|---:|---|
| AUTH_REQUIRED | 401 | 未登录或 token 缺失 |
| TOKEN_EXPIRED | 401 | token 过期 |
| PERMISSION_DENIED | 403 | 角色无权限 |
| USER_DISABLED | 403 | 用户被禁用 |

### 数据窗口

| code | HTTP | 说明 |
|---|---:|---|
| CASE_NOT_FOUND | 404 | case 不存在 |
| WINDOW_NOT_FOUND | 404 | window 不存在 |
| WINDOW_NOT_EDITABLE | 409 | window 状态不可编辑 |
| WINDOW_LOCKED | 409 | window 被其他 session 锁定 |
| GRID_SHAPE_MISMATCH | 422 | 网格 shape 不匹配 |
| FILE_NOT_FOUND | 404 | 原始或归档文件不存在 |
| DIMENSION_MISMATCH | 422 | 变量维度不一致 |
| QPF_NEGATIVE_WARNING | 200/409 | qpf 差分存在负值，按配置决定是否阻止 |

### 编辑

| code | HTTP | 说明 |
|---|---:|---|
| SESSION_NOT_FOUND | 404 | session 不存在 |
| SESSION_NOT_EDITING | 409 | session 不是 editing |
| SESSION_EXPIRED | 409 | session 过期 |
| PREVIEW_NOT_FOUND | 404 | preview 不存在 |
| PREVIEW_EXPIRED | 409 | preview 过期或被覆盖 |
| PREVIEW_SESSION_MISMATCH | 400 | preview 不属于 session |
| MASK_OUT_OF_BOUNDS | 422 | mask 超出网格范围 |
| INVALID_OPERATION_PARAM | 422 | 编辑参数非法 |
| INVALID_PTYPE | 422 | ptype 非 0/1/2/3 |

### 版本与发布

| code | HTTP | 说明 |
|---|---:|---|
| VERSION_NOT_FOUND | 404 | version 不存在 |
| VERSION_STATUS_CONFLICT | 409 | 当前状态不允许该操作 |
| VERSION_BASE_OUTDATED | 409 | session 基线不是最新版本 |
| RELEASE_CONFLICT | 409 | 发布冲突 |
| VERSION_SAVE_FAILED | 500 | 版本保存失败 |

### 绘图与复盘

| code | HTTP | 说明 |
|---|---:|---|
| REVIEW_NOT_FOUND | 404 | review 不存在 |
| TEMPLATE_NOT_FOUND | 404 | 模板不存在 |
| TEMPLATE_VALIDATION_FAILED | 422 | 模板配置非法 |
| PLOT_TASK_NOT_FOUND | 404 | 绘图任务不存在 |
| PLOT_TASK_TIMEOUT | 500 | 绘图超时 |
| PLOT_TASK_FAILED | 500 | 绘图失败 |
| REQUIRED_FIELD_MISSING | 422 | 必需复盘字段缺失 |

### 配置与系统

| code | HTTP | 说明 |
|---|---:|---|
| CONFIG_NOT_FOUND | 404 | 配置不存在 |
| CONFIG_VALIDATION_FAILED | 422 | 配置校验失败 |
| FILE_WRITE_FAILED | 500 | 文件写入失败 |
| STORAGE_NOT_ENOUGH | 507 | 存储空间不足 |
| INTERNAL_ERROR | 500 | 未分类内部错误 |

## 16.11 状态转移校验示例

```python
def submit_version(version):
    if version.status != 'draft':
        raise DomainError('VERSION_STATUS_CONFLICT')
    version.status = 'submitted'
```

```python
def release_version(version):
    if version.status != 'approved':
        raise DomainError('VERSION_STATUS_CONFLICT')
    supersede_active_release(version.window_id)
    version.status = 'released'
```

## 16.12 前端错误展示建议

| 错误码 | 前端行为 |
|---|---|
| WINDOW_NOT_EDITABLE | 弹窗显示原因，禁止进入编辑 |
| SESSION_EXPIRED | 提示 session 过期，回到窗口选择 |
| VERSION_BASE_OUTDATED | 提示存在新版本，提供重新加载 |
| PREVIEW_EXPIRED | 清空 preview，要求重新预览 |
| PLOT_TASK_FAILED | 展示 error_log 下载入口 |
| PERMISSION_DENIED | 跳转 403 或禁用按钮 |
