## 14. 复盘系统设计

### 14.1 复盘系统定位

复盘系统不参与人工编辑，只负责把编辑结果和 IFS 多要素诊断场组织成绘图输入，并调用用户提供的绘图代码生成组合图。

### 14.2 当前时段与前后 24h 定义

当前编辑窗口：

```text
current_window = [start_lead, end_lead]
```

前 24h：

```text
prev24_window = [start_lead - 24, start_lead]
```

后 24h：

```text
next24_window = [end_lead, end_lead + 24]
```

示例：

```text
当前窗口：+024h ~ +048h，24h累计
前24h：+000h ~ +024h
后24h：+048h ~ +072h
```

如果前后 24h 数据不存在，则复盘图对应区域标注“无数据”，并在 manifest 中记录缺失。

### 14.3 IFS 诊断场时间取样

累计降水是一个时间段，而高度场、风场、温度场、湿度场通常是某个有效时刻的预报场。因此需要制定时间取样策略。

建议预留配置：

```json
{
  "review_time_policy": {
    "current_window": ["start", "middle", "end"],
    "prev24_window": ["end"],
    "next24_window": ["start"],
    "default_pressure_field_time": "middle"
  }
}
```

首版可简化为：

```text
当前窗口：取中间时刻和结束时刻
前24h：取结束时刻
后24h：取开始时刻
```

具体取样规则可在接入绘图代码后再精确调整。

### 14.5 绘图模板管理

建议后续支持多个复盘模板：

| window_prev_next_v1 | 当前时段与前后 24h 图 | 前24h、当前、后24h |

### 15.1 数据格式建议

| 数据类型 | 建议格式 | 说明 |
|---|---|---|
| 小规模原型 | `.npy` / `.npz` | 开发快，便于测试 |
| 业务归档 | NetCDF | 便于保存维度、变量、单位和元数据 |
| 大规模复盘样本库 | Zarr | 便于多时效、多版本、局部读取 |
| 图片产品 | PNG | 便于展示和归档 |
| 操作日志 | JSONL | 一行一个操作，便于追加和分析 |
| 统计结果 | CSV | 便于业务人员查看 |

首版建议：

```text
原型阶段：NPZ + PNG + JSONL + CSV
业务归档阶段：NetCDF + PNG + JSONL + CSV
```
