# 18. 测试用例与验收数据设计

## 18.1 设计目标

测试设计覆盖数据接入、编辑引擎、版本审核发布、复盘绘图、权限审计和前端 E2E。开发过程中每个模块必须至少具备单元测试和集成测试。

## 18.4 集成测试

### test_data_scan_create_windows

流程：

```text
1. 准备 normal_case
2. POST /api/data/scan
3. GET /api/windows
4. 校验窗口数量、status、qc_status
```

### test_session_preview_apply_save

流程：

```text
1. start session
2. load session
3. preview qpf increase
4. apply preview
5. save version
6. 校验 edit_version、edit_operation、文件产物
```

### test_undo_redo

```text
1. apply 两次操作
2. undo
3. 校验状态等于第一步后
4. redo
5. 校验状态恢复
```

### test_submit_approve_release

```text
1. 保存 draft
2. submit
3. approve
4. release
5. 校验 release_product active
6. 发布新版本，旧版本 superseded
```

### test_review_generate_partial_success

```text
1. 准备缺 optional IFS 变量
2. generate review
3. 任务 partial_success
4. missing_fields 有记录
5. 组合图生成占位面板
```

### test_export_review_package

```text
1. 对 released version 导出
2. zip 中包含 metadata、fields、images、manifest、operation_log
```

## 18.5 前端 E2E

使用 Playwright 或 Cypress。

### E2E-01 登录和权限

```text
viewer 登录 → 编辑按钮隐藏
forecaster 登录 → 可编辑但无审核/发布按钮
reviewer 登录 → 可审核/发布
admin 登录 → 可管理用户和配置
```

### E2E-02 编辑闭环

```text
登录 → 选择 case/window → start session → 画 polygon → qpf increase → preview → apply → undo → redo → save
```

### E2E-03 审核发布

```text
forecaster 保存并提交 → reviewer 审核通过 → release → review center 显示 released
```

### E2E-04 复盘中心

```text
打开复盘中心 → 查询 case → 查看窗口版本 → 查看复盘图 → 重新生成 → 导出 zip
```

## 18.6 异常测试

| 场景 | 期望 |
|---|---|
| tp_end 缺失 | window invalid |
| ptype 全缺 | window invalid |
| ptype 部分缺 | window partial |
| qpf 负值 | qc_status=warn，negative_count > 0 |
| preview 过期 | PREVIEW_EXPIRED |
| session 过期 | SESSION_EXPIRED |
| 版本状态错误 | VERSION_STATUS_CONFLICT |
| 无权限发布 | PERMISSION_DENIED |
| 绘图超时 | PLOT_TASK_TIMEOUT，可 retry |
| 存储不足 | STORAGE_NOT_ENOUGH |

## 18.7 性能测试

| 操作 | 目标 |
|---|---:|
| 单窗口 qpf/ptype 构建 | ≤ 5s |
| 窗口列表查询 | ≤ 1s |
| 多边形 preview | ≤ 3s |
| 笔刷 apply | ≤ 1s |
| undo/redo 重放 50 步 | ≤ 5s |
| 保存版本 | ≤ 10s，不含复杂绘图 |
| 绘图任务入队 | ≤ 1s |
| 复盘包导出 | 视包大小，需进度提示 |

## 18.8 验收数据要求

最终验收至少提供：

```text
1. 一个完整正常 case，含 000-240h tp/ptype。
2. 一个 partial case，缺少部分 ptype。
3. 一个 invalid case，缺关键 tp。
4. 一个 negative case，存在 qpf 负值。
5. 一个冬季相态 case，必须包含 0/1/2/3 四类 ptype。
6. 一个复盘 IFS case，含 500/700/850 高度、风、温湿。
```
